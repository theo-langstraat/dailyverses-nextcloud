

# 🏷️ **Beschikbare Conventional Commit labels**

## ✨ **Feature‑ontwikkeling**
- **feat** — nieuwe functionaliteit, nieuwe capability, nieuwe optie

## 🐛 **Bugfixes**
- **fix** — bug opgelost, fout hersteld, crash voorkomen

## 🔧 **Code‑herstructurering**
- **refactor** — code herschreven zonder functionaliteit te veranderen

## 🎨 **Code‑stijl**
- **style** — formatting, whitespace, puntkomma’s, geen code‑logica

## 📚 **Documentatie**
- **docs** — README, comments, changelog, documentatie

## 🧪 **Tests**
- **test** — nieuwe tests, testfixes, teststructuur

## 🧹 **Chores / onderhoud**
- **chore** — onderhoud, scripts, dependencies, CI, tooling

## 🚀 **Performance**
- **perf** — performance‑optimalisaties

## 🛠️ **Build‑systeem**
- **build** — build scripts, npm, composer, bundlers

## 🔁 **Continuous Integration**
- **ci** — GitHub Actions, pipelines, CI‑config

## ♻️ **Reverts**
- **revert** — commit terugdraaien

---

# 🎯 **Praktische voorbeelden voor jouw project**

Voor jouw Nextcloud‑app zijn dit de meest gebruikte:

```
feat: add RTL auto-detection
fix: correct JSON parsing in settings
refactor: simplify load() logic
docs: update README_RELEASE.md
chore: update release scripts
build: adjust Vite config for production
ci: add GitHub Actions release workflow
```

---

# 🧠 Bonus: scopes (optioneel)

Je kunt commits nóg specifieker maken:

```
fix(settings): correct admin form validation
feat(widget): add verse fade-in animation
refactor(api): simplify axios wrapper
docs(release): update changelog instructions
```

Scopes zijn **vrij te kiezen**.
